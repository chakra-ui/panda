# Generated types: legacy JS generator vs Rust codegen

> First measured comparison of the generated **type graph**. Both paths consume the same fixture configs
> (`fixtures/generated-types/configs/*.json`): the Rust side via `Project::generate_artifacts`, the legacy side via
> `@pandacss/node` codegen with `eject: true`. Both emit `.d.ts` declarations. An identical `usage.ts` imports
> `./styled-system/types`, builds a recursive `SystemStyleObject`, and is checked with `tsc --extendedDiagnostics`
> (`skipLibCheck: true`). Runtime helpers (`cva`/`sva`/recipe runtimes) are excluded ‚Äî WIP, not type-cost.

## TL;DR

Rust is **dramatically cheaper on every metric** for the same usage and autocomplete. The headline is instantiations:
**~130√ó fewer** (~20 vs ~2,600), because every native CSS property shares one cached `ConditionalValue<CssValue>` and
`skipLibCheck` skips the declaration bodies ‚Äî only the user's own `css({‚Ä¶})` shape drives instantiation.

| Fixture | Instantiations (rust / legacy) |        Œî | Types (rust / legacy) |    Œî |      Memory K |    Œî | Check ms |    Œî |
| ------- | -----------------------------: | -------: | --------------------: | ---: | ------------: | ---: | -------: | ---: |
| small   |                      20 / 2603 | **‚àí99%** |            187 / 2427 | ‚àí92% | 65790 / 86888 | ‚àí24% |  30 / 50 | ‚àí40% |
| medium  |                      20 / 2639 | **‚àí99%** |            212 / 2487 | ‚àí91% | 66046 / 83584 | ‚àí21% |  20 / 50 | ‚àí60% |
| large   |                      20 / 2648 | **‚àí99%** |            534 / 2988 | ‚àí82% | 66969 / 89119 | ‚àí25% |  20 / 50 | ‚àí60% |
| strict  |                      26 / 2575 | **‚àí99%** |            215 / 1932 | ‚àí89% | 65803 / 84376 | ‚àí22% |  20 / 50 | ‚àí60% |

`jsx-{all,minimal,none}` match `medium` on `Types`/instantiations (the Rust codegen doesn't emit distinct JSX artifacts
yet). Strict is marginally higher (26 vs 20) ‚Äî `WithEscapeHatch`/`OnlyKnown` wrappers over the configured literals
instantiate on use ‚Äî still ‚àí99% vs legacy.

These numbers reflect the final architecture: **our own csstype** (one shared `CssValue`, no vendored keyword unions),
the core support surfaces **merged into one `types/system.d.ts`**, and a **single shared property registry**
(`pandacss_shared::css_properties`) feeding both the extractor and the codegen. Memory fell to ~66-67MB once the
vendored csstype `Property` namespace was dropped entirely.

## Measurement: why this must be `.d.ts` + skipLibCheck

This is the whole story, and the first cut of this harness got it wrong. Panda ships the styled-system as **declaration
files**, and every Panda project runs **`skipLibCheck`** ‚Äî so TypeScript never deep-checks `styled-system/**/*.d.ts`.
The cost a user actually pays is on-demand instantiation when they write `css({‚Ä¶})`, not checking the generated lib.

The first run emitted the Rust side as `.ts` _source_ (`format: Ts`) while legacy shipped `.d.ts`. That defeated
skipLibCheck on the Rust side only, so the (then-vendored) csstype's ~600-union `Property` namespace was fully checked
and counted ‚Äî making Rust look ~2√ó _worse_ on `Types`. Emitting Rust as `.d.ts` (`format: Js`, matching legacy and real
output) is what these numbers reflect.

### How we got from ‚àí98% to ‚àí99% (kept as a record)

The early cuts indexed a vendored, terse csstype (`CssProperties["display"]`) ‚Äî fast, but ~50 instantiations and a heavy
`Property` namespace in the wasm binary. Two dead ends along the way: a mapped `Exclude<keyof CssProperties, ‚Ä¶>` fallback
spiked to **2,647** instantiations (an `Exclude` over ~850 keys), and an `export type * ‚Üí export type { CssProperties }`
barrel tweak proved **perf-neutral** (kept only as API hygiene). The win came from replacing the vendored csstype with
**our own `CssValue`-based csstype** (every native prop shares one cached type), which dropped instantiations to ~20 and
let the `Property` namespace ‚Äî and the 136KB asset ‚Äî be deleted outright (the ‚àí24% memory).

## Recipe / pattern / cva (spot-checked separately)

The main table's `usage` only exercises `SystemStyleObject`. A separate Rust-side check on the `recipes` fixture
(`button` recipe + `stack` pattern + `cva` + `RecipeVariantProps`) confirms those user-facing types are **correct, live,
and cheap**. (Absolute numbers below are from the earlier csstype-indexed cut ‚Äî base is now ~20, not 49 ‚Äî but the
bounded-delta / no-blowup conclusion is unchanged.)

| usage                              | Types | Instantiations |
| ---------------------------------- | ----: | -------------: |
| base (`SystemStyleObject` only)    |   658 |             49 |
| heavy (+ recipe + pattern + `cva`) |   849 |        **228** |

A bounded +179 instantiations for a full recipe/pattern/cva exercise ‚Äî no blowup (v1's equivalent is far higher given
its `DistributiveOmit`/`Assign` recipe machinery). Negative tests confirm liveness: `button({ size: 'huge' })` and
`stack({ direction: 'diagonal' })` are both rejected against the precise variant/prop unions. Legacy-vs-Rust isn't
table-compared here because the flat (`styled-system/index`) vs nested (`styled-system/css`) barrel layouts give
different import paths; a cross-layout comparison is future work.

## Caveats

- **Main table is type-graph only.** Its `usage.ts` imports `./styled-system/types` and builds a `SystemStyleObject`
  (tokens, conditions, nesting, an arbitrary selector); values are real (`color: 123` is rejected). Recipe/pattern/cva
  are covered in the section above, not the main table.
- **`eject: true`** on the legacy side (no base preset) keeps the utility/token surface equal to the fixture and to the
  Rust side. Fair comparison; not representative of a full preset's absolute size.
- **JSX not captured.** No distinct Rust JSX artifacts yet, so the `jsx-*` fixtures equal `medium`. Add a JSX usage once
  those land to measure `jsxStyleProps`.
