# Tokens

Design tokens are the platform-agnostic way to manage design decisions in your application or website.

A design token is a collection of attributes that describe any fundamental/atomic visual style. Each attribute is a
key-value pair.

> Design tokens in Panda are largely influenced by the [W3C Token Format](https://tr.designtokens.org/format/).

Tokens are defined in the `panda.config.(ts|mjs)` file, and can contain the following categories: colors, fonts, sizes,
shadows, borders, gradients, and spacings.

## Raw Tokens

A raw token consist of the following properties:

- `value`: The value of the token. This can be any valid CSS value.
- `description`: An optional description of the token.

```js
export default {
  tokens: {
    colors: {
      danger: { value: '#EE0F0F' },
    },
  },
}
```

Now you can use this color when writing styles.

```js
css({ color: 'danger' })
```

## Semantic or Alias Tokens

An alias token is a token that reference to another token or defines a more specific context. It is often used to
communicate the intended purpose of a token independent of the actual/primitive value of the token.

Semantic tokens are defined using the `{}` syntax

### Basic example

A quick example:

- `red` and `green` are raw tokens that define the color red and blue.
- `danger` and `success` are semantic tokens that reference to the `red` and `blue` tokens.

```js
export default {
  tokens: {
    colors: {
      red: { value: '#EE0F0F' },
      green: { value: '#0FEE0F' },
    },
  },
  semanticTokens: {
    colors: {
      danger: { value: '{colors.red}' },
      success: { value: '{colors.green}' },
    },
  },
}
```

### Using conditions

Semantic tokens can also be changed based on the [conditions](./conditional-styles). For example, if you want to a color
to change based on light or dark mode automatically.

> NOTE ðŸš¨: The conditions used in semantic tokens most be an at-rule or a parent selector. To see the list of supported
> conditions, see [Conditional Styles](./conditional-styles).

```js
export default {
  // ...
  semanticTokens: {
    colors: {
      danger: { value: { base: '{colors.red}', _dark: '{colors.darkred}' } },
      success: { value: { base: '{colors.green}', _dark: '{colors.darkgreen}' } },
    },
  },
}
```

### Designing multiple themes

With semantic tokens, you can add support for multiple themes in your application.

Let's say your panda configuration includes the following conditions:

```js
export default {
  conditions: {
    light: '@media (prefers-color-scheme: light)',
    dark: '@media (prefers-color-scheme: dark)',
    pinkTheme: '[data-theme=pink] &',
    blueTheme: '[data-theme=blue] &',
  },
}
```

You can defined semantic tokens for each theme, like so:

```js
export default {
  // ...
  semanticTokens: {
    colors: {
      text: {
        pinkTheme: { _light: '{colors.pink}', _dark: '{colors.darkpink}' },
        blueTheme: { _light: '{colors.blue}', _dark: '{colors.darkblue}' },
      },
    },
  },
}
```

Now, you can use the `text` token in your styles, and it will automatically change based on the theme and the color
scheme.

```jsx
// use pink theme
<html data-theme="pink">
  <body>
    <h1 css={{ color: 'text' }}>Hello World</h1>
  </body>
</html>
```

## Automatic Documentation

Panda provide automatic documentation for tokens defined in your project.

You can see the documentation by running `panda docs` in your project directory.

```bash
pnpm panda docs
```
