# Writing Styles

Panda generates the utilities you need to style your components with confidence.

## Atomic styles

The `css` function is used to write atomic styles. It accepts a style object of both native and user-defined properties,
and returns a string.

```js
import { css } from './styled-system/css'

const btnClass = css({
  backgroundColor: 'gainsboro',
  borderRadius: '9999px',
  fontSize: '13px',
  padding: '10px 15px'
})

// => bg-gainsboro br-9999px fs-13px p-10px_15px
```

## Conditional Styles

Conditional styles are styles that change based on media query, selector or at-rule. They can be changed by providing
nested objects or modifying the property to use a conditional object.

Here's an example of how to modify a property using nested objects.

```js
css({
  color: 'red',
  // change the color based on "sm" breakpoint
  sm: {
    color: 'pink'
  }
})
```

Here's an example of how to modify a property using a conditional object.

```js
css({
  color: { base: 'red', sm: 'pink' }
})
```

Looks pretty similar, right? The difference is that the inline conditional object approach gives a more flexibility.

```js
css({
  color: { base: 'red', _hover: 'pink', _active: 'blue' }
})
```

Conditions can be nested as well.

```js
css({
  fontSize: {
    base: { _ltr: '16px', _rtl: '18px' },
    md: { _ltr: '18px', _rtl: '20px' }
  }
})
```

## Nested Styles

Panda provides different ways of nesting style declarations.

> **Important:** Panda requires that you use the "&" selector to create nested styles.

For simple scenarios, you might find the grouped "selectors" object too verbose, you can apply simple pseudo or
attribute selectors directly in the css object.

```jsx
<div
  className={css({
    bg: 'red.400',
    '&:hover': {
      bg: 'orange.400'
    }
  })}
/>
```

### Using Pseudo Props

CSS styles can be nested using the css conditions in the panda config.

```jsx
<div
  className={css({
    bg: 'red.400',
    _hover: {
      bg: 'orange.400'
    },
    sm: {
      fontSize: '50px'
    }
  })}
/>
```

### Caveats

To ensure nesting works as expect, it's required to use the nesting syntax `&` when selecting siblings, parent,
children, etc.

Here are some styles that won't work as expected.

```js
css({
  bg: 'red.500',
  // ‚ùå won't work. It should be `& span`
  span: {
    color: 'pink.500'
  }
})
```

## Global styles

To write global styles in your app, add it to your `panda.config.ts`

```js
// panda.config.ts
export default defineConfig({
  globalCss: {
    'html, body': {
      margin: 0,
      padding: 0
    }
  }
})
```

## Important styles

Applying important styles works just like CSS

```js
css({
  color: 'red !important'
})
```

You can also apply important using just the exclamation syntax `!`

```js
css({
  color: 'red!'
})
```
