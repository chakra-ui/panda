Use Panda's command-line interface (CLI) to generate the styled-system output and CSS from a terminal.

## Commands

| Command           | Description |
| ----------------- | ----------- |
| `panda init`      | Scaffold `panda.config.ts` and run the first codegen. |
| `panda dev`       | Watch project files and rebuild the generated system and CSS. |
| `panda build`     | Generate the styled-system output and CSS once. Bare `panda` runs the same build. |
| `panda check`     | Check generated files without writing. Use this in CI. |
| `panda info`      | Print project and compiler info. |
| `panda doctor`    | Check config loading and compiler diagnostics. |
| `panda debug`     | Write bug-report artifacts under `<outdir>/debug`. |
| `panda codegen`   | Advanced: generate the styled-system output only. |
| `panda cssgen`    | Advanced: generate CSS only. |
| `panda buildinfo` | Advanced: emit a portable design-system build manifest. |

## Shared Flags

| Flag | Description |
| ---- | ----------- |
| `--cwd <dir>` | Current working directory. |
| `--config, -c <path>` | Path to the Panda config file. |
| `--json` | Print the command result as JSON. |
| `--format <format>` | Diagnostic output format: `human`, `pretty`, `json`, or `github`. |
| `--log-level <level>` | Output level: `silent`, `error`, `warn`, `info`, or `debug`. |
| `--max-warnings <n>` | Exit with failure when warning diagnostics exceed this count. |
| `--logfile <file>` | Tee human output to a log file. JSON output is not written to the log file. |
| `--trace` | Enable compiler tracing. |
| `--trace-output <format>` | Trace output: `fmt` or `chrome-json`. |
| `--trace-file <file>` | Trace output file for `chrome-json` tracing. |

`--silent`, `--quiet`, and `--verbose` are replaced by `--log-level`.

## init

Initialize Panda in a project.

```bash
pnpm panda init
pnpm panda init --postcss
pnpm panda init --outdir styled-system
```

| Flag | Description |
| ---- | ----------- |
| `--force, -f` | Overwrite an existing config file. |
| `--postcss, -p` | Emit a PostCSS config file. |
| `--gitignore` / `--no-gitignore` | Update `.gitignore` with the output directory. |
| `--codegen` / `--no-codegen` | Run codegen after setup. |
| `--out-extension <ext>` | Generated runtime file extension: `ts`, `js`, or `mjs`. |
| `--outdir <dir>` | Output directory for generated files. |
| `--jsx-framework <framework>` | JSX framework to use. |
| `--syntax <syntax>` | CSS syntax preference: `template-literal` or `object-literal`. |
| `--strict-tokens` | Set `strictTokens` to `true`. |
| `--install` / `--no-install` | Install default presets when possible. |

## build

Generate the styled-system output and CSS once.

```bash
pnpm panda build

# Bare panda is also a build
pnpm panda

# Write generated output and CSS under another directory
pnpm panda build --outdir system
```

| Flag | Description |
| ---- | ----------- |
| `--outdir <dir>` | Output directory for generated files and the default CSS file. |
| `--outfile, -o <file>` | Output file for extracted CSS. |
| `--splitting` | Emit split CSS files. |
| `--clean` | Clean the output directory before generating. |
| `--check` | Check generated files without writing. Prefer `panda check` for CI. |
| `--watch, -w` | Watch files and rebuild. Prefer `panda dev` for watch mode. |
| `--watch-debounce <ms>` | Watch rebuild debounce in milliseconds. |

## dev

Watch project files and rebuild the generated system and CSS.

```bash
pnpm panda dev
pnpm panda dev --outdir system
```

`panda dev` uses the same build flags as `panda build`, except watch mode is enabled by the command.

## check

Check generated files without writing.

```bash
pnpm panda check
pnpm panda check --format github --max-warnings 0
```

`panda check` uses the same output flags as `panda build`, but write/watch-only flags are hidden from help.

## info

Print project and compiler information.

```bash
pnpm panda info
pnpm panda info --json
```

The result includes config path, source count, watch directories, artifact IDs, condition count, token category count,
and utility count.

## doctor

Check config loading and compiler diagnostics.

```bash
pnpm panda doctor
pnpm panda doctor --format github --max-warnings 0
```

Use this command when you want a fast setup and diagnostics health check without writing generated files.

## debug

Write bug-report artifacts.

```bash
pnpm panda debug
pnpm panda debug --outdir panda-debug
pnpm panda debug --dry
pnpm panda debug --only-config
```

The debug dump includes:

- `system-info.json`
- `config.json`
- `<file>.extract.json`
- `styles.css`

| Flag | Description |
| ---- | ----------- |
| `--outdir <dir>` | Debug output directory. Defaults to `<styled-system>/debug`. |
| `--dry` | Print the dump to stdout instead of writing files. |
| `--only-config` | Only dump the resolved config. |

## codegen

Generate the styled-system output only.

```bash
pnpm panda codegen
pnpm panda codegen --clean
pnpm panda codegen --check
```

| Flag | Description |
| ---- | ----------- |
| `--watch, -w` | Watch files and rebuild. |
| `--outdir <dir>` | Output directory for generated files. |
| `--clean` | Clean the output directory before generating. |
| `--watch-debounce <ms>` | Watch rebuild debounce in milliseconds. |
| `--check` | Check generated files without writing. |

## cssgen

Generate CSS only.

```bash
pnpm panda cssgen
pnpm panda cssgen --outfile dist/styles.css
pnpm panda cssgen --splitting
pnpm panda cssgen --check
```

| Flag | Description |
| ---- | ----------- |
| `--watch, -w` | Watch files and rebuild. |
| `--outfile, -o <file>` | Output file for extracted CSS. |
| `--splitting` | Emit split CSS files. |
| `--watch-debounce <ms>` | Watch rebuild debounce in milliseconds. |
| `--check` | Check generated CSS without writing. |

## buildinfo

Emit a portable design-system build manifest.

```bash
pnpm panda buildinfo
pnpm panda buildinfo --outfile dist/panda.buildinfo.json
pnpm panda buildinfo --minify
```

| Flag | Description |
| ---- | ----------- |
| `--outfile, -o <file>` | Output path. Defaults to `<outdir>/panda.buildinfo.json`. |
| `--panda <range>` | Peer Panda version range to stamp into the artifact. Defaults to `*`. |
| `--minify, -m` | Minify the generated JSON. |

## Removed Commands

`panda inspect` and `panda validate` are removed. Use `panda info` and `panda doctor`.

Legacy commands such as `studio`, `analyze`, `ship`, and `emit-pkg` are not part of the v2 CLI surface.
