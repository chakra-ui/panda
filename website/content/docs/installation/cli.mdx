This guide explains how to use Panda CSS through the Panda CLI instead of a framework-specific integration. The installed package scans your project files and generates the required CSS utilities automatically.

<Steps>

### Installation of Panda

This section explains how to install Panda using the CLI.

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  {/* <!-- prettier-ignore-start --> */}
  <Tab value="pnpm">
    ```bash
    pnpm install -D @pandacss/dev
    
    # Initialize Panda CSS
    pnpm panda init
    ```
  </Tab>

  <Tab value="npm">
    ```bash
    npm install -D @pandacss/dev
    
    # Initialize Panda using npx without installing it globally
    npx panda init
    ```
  </Tab> 
  
  <Tab value="yarn">
    ```bash
    yarn add -D @pandacss/dev

    # Run the Panda initialization command
    yarn panda init
    ```
  </Tab>
  
  <Tab value="bun">
    ```bash
    bun add -D @pandacss/dev
    
    # Initialize Panda in the project
    bun panda init
    ```
  </Tab>
  {/* <!-- prettier-ignore-end --> */}
</Tabs>


### Setting Up File Paths for Style Generation

Add these paths to all of your JavaScript or TypeScript code where you intend to use panda. 
This allows Panda to scan your code and generate the necessary CSS utilities.


```js {5}
import { defineConfig } from '@pandacss/dev'

export default defineConfig({
  preflight: true,
  include: ['./src/**/*.{ts,tsx,js,jsx}', './pages/**/*.{ts,tsx,js,jsx}'],
  exclude: [],
  outdir: 'styled-system'
})
```

### Update package.json scripts

Open your `package.json` file and update the `scripts` section.
The `package.json` file is the central configuration for your project 
it keeps track of dependencies, project metadata, and custom commands 
that you can run from the terminal.

```diff {3}
{
  "scripts": {
+    "prepare": "panda codegen",
  }
}
```

The `prepare` script that will run codegen after dependency installation. Read more about
[codegen](/docs/references/cli#panda-codegen) in the CLI section.

> This step ensures that the panda output directory is regenerated after each dependency installation. So you can add
> the output directory to your `.gitignore` file and not worry about it.

### Import the generated CSS

For each Panda run, it emits the generated CSS at the `styled-system/styles.css` file path. Import this file at the root
component of your project.

```jsx {1}
import './styled-system/styles.css'

export function App() {
  return <div>Page</div>
}
```

### Start the Panda build process

Run the CLI tool to scan your JavaScript and TypeScript files for style properties and call expressions.

### Start using Panda

Use the generated style utilities in your code and panda will extract them to the generated CSS file. Then run your
build process.

```jsx
import { css } from './styled-system/css'

export function App() {
  return <div className={css({ bg: 'red.400' })} />
}
```

</Steps>

## Troubleshooting

If you're not getting import autocomplete in your IDE, you may need to include the `styled-system` directory in your
`tsconfig.json` file:

```json filename="tsconfig.json"
{
  // ...
  "include": ["src", "styled-system"]
}
```
